<objective>
Fix /podcast/archive so it lists all past episodes — including pre-Blob episodes that only have a Redis podcast-script:{date} key — and shows a conditional play button only where a cached MP3 exists.

Purpose: listPodcastDates() currently reads only Vercel Blob. Episodes generated before Blob was activated (session 1-5) have a podcast script in Redis but no MP3 in Blob, so they are invisible in the archive. Separately, the archive page links every row to the episode — but without audio, that page silently renders without a player, confusing subscribers.

Output: New listPodcastDatesWithStatus() in lib/podcast-storage.ts that unions Blob dates (hasAudio: true) and Redis script dates (hasAudio: false), plus updated archive page that renders a play icon link only for hasAudio rows.
</objective>
