  <files>lib/storage.ts</files>
  <action>
Add a new exported async function `backfillQuizIndex(): Promise<number>` at the bottom of the Quiz storage section (after `listQuizDates()`).

The function must:
1. Return 0 immediately if `useRedis()` is false (no Redis in dev — no-op)
2. Call `getRedis()` to get the Redis client
3. Use cursor-based scan to iterate all Redis keys matching `quiz:*`
   - Start cursor at 0
   - Call `redis.scan(cursor, { match: 'quiz:*', count: 100 })`
   - Destructure as `const [nextCursor, keys] = ...`
   - Coerce cursor: `cursor = Number(nextCursor)` — Upstash returns cursor as a string, comparison against 0 fails without this coercion
   - Continue loop while `cursor !== 0`
4. For each key, match against the regex `/^quiz:(\d{4}-\d{2}-\d{2})$/` — this filters out `quiz:index` itself and any other non-date quiz keys
5. For each matching date, call:
   ```
   await redis.zadd('quiz:index', {
     nx: true,
     score: new Date(date).getTime(),
     member: date,
   });
   ```
   The `nx: true` flag makes the backfill idempotent — existing entries are never overwritten, safe to run multiple times.
6. Increment a `count` variable for each successful zadd (whether or not the entry was new)
7. Return `count` (total quiz keys found matching the date pattern, not just newly added)

Add a JSDoc comment: "One-time backfill — scans all quiz:{date} Redis keys and writes missing entries into quiz:index sorted set. Safe to run multiple times (nx flag). Returns the number of date keys found."

Do NOT modify any existing functions. Add only the new export.
