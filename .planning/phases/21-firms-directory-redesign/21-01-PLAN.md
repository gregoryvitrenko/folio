- Tier filter tabs: vertical list of buttons. One per tier plus "All" at top.

  - Use existing search markup but apply `rounded-chrome` instead of `rounded-sm`

**Right column:** Render firms grouped by tier (same `TIER_ORDER` logic). When `activeTier !== 'All'` and no search query, only render the active tier section — hide the rest. Keep the existing tier section header treatment (full-width rule + tier name). Remove the jump links (now replaced by sidebar filter tabs). Keep the result count text when search query is active.

**Simplest mobile approach:** Use responsive classes. The sidebar `<aside>` has `className="lg:w-52 lg:shrink-0 lg:sticky lg:top-24"`. Its inner tab list has `className="flex gap-2 overflow-x-auto pb-2 lg:flex-col lg:overflow-x-visible lg:pb-0"`. This gives horizontal scrolling tabs on mobile and vertical tabs on desktop automatically.
