  <name>Task 1: Add HMAC helpers to lib/email.ts and create unsubscribe API route</name>
  <files>lib/email.ts, app/api/unsubscribe/route.ts</files>
  <action>
**In lib/email.ts**, add two new exported functions at the top of the file (below imports):

```typescript
import crypto from 'crypto';
```

1. `signUnsubscribeToken(email: string, secret: string): string` — returns `crypto.createHmac('sha256', secret).update(email).digest('hex')`.

2. `buildUnsubscribeUrl(email: string, siteUrl: string): string` — reads `CRON_SECRET` from env, calls `signUnsubscribeToken`, returns `${siteUrl}/api/unsubscribe?email=${encodeURIComponent(email)}&sig=${sig}`.

Do NOT modify `sendWeeklyDigest` signature yet — Plan 02 handles that.

**Create app/api/unsubscribe/route.ts** with a single `GET` handler:
1. Read `email` and `sig` query params from the URL.
2. Validate: if `email` or `sig` missing, or `sig` length !== 64, redirect to `/unsubscribe?error=invalid`.
3. Compute expected HMAC using `CRON_SECRET` env var.
4. Compare using `crypto.timingSafeEqual` wrapped in try/catch (return false on any error — handles malformed hex).
5. If invalid, redirect to `/unsubscribe?error=invalid`.
6. If valid and Redis is configured (`UPSTASH_REDIS_REST_URL` + `UPSTASH_REDIS_REST_TOKEN` both set), set `email-opt-out:${email.toLowerCase()}` to `'1'` with no TTL.
7. Redirect to `/unsubscribe?success=1`.

Use the same `getRedis()` pattern as `lib/subscription.ts` (require @upstash/redis inline). Import `crypto` from Node built-in.
  </action>
