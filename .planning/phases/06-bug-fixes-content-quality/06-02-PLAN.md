<objective>
Fix the quiz caching bug where listQuizDates() returns all briefing dates (most without cached quiz data), causing the /quiz page to list dates that lead to empty screens.

Purpose: The "Available" list in /quiz misleads students — clicking a date shows no questions. The root cause is that listQuizDates() in Redis mode uses the briefing index, not a quiz-specific index. The fix adds a quiz:index sorted set (mirroring the existing briefing:index pattern) and removes the hardcoded ?? 18 fallback badge count.
Output: lib/storage.ts with quiz:index tracking, app/quiz/[date]/page.tsx with accurate question count.
</objective>

function fsListQuizDates(): string[] {
  return fs.readdirSync(DATA_DIR)
    .filter((f) => f.endsWith('-quiz.json'))
    .map((f) => f.replace('-quiz.json', ''))
    .filter((d) => /^\d{4}-\d{2}-\d{2}$/.test(d))
    .sort((a, b) => b.localeCompare(a));
}
