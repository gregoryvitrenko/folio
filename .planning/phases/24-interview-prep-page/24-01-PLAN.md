**File structure pattern (server + client in one file):**
The server component is the default export (async, calls requireSubscription, passes serialised data as props). The client component is a named export above it, marked 'use client' at the top of the file.

Wait — Next.js App Router does NOT allow mixing 'use client' and async server components in the same file. Use the standard pattern: create the client component inline by passing the data down via a separate client component file, OR more simply, make the entire page a client component and call requireSubscription via a server action / middleware check.

**Correct approach for this page:** The page is a server component that calls requireSubscription, then renders a client component. Put the client component in a separate file: `app/interview/InterviewBank.tsx`. The server page file imports it.

**File 2: app/interview/page.tsx** — Server component

```typescript
import { requireSubscription } from '@/lib/paywall';
import { PRIMERS } from '@/lib/primers-data';
import { getTodayDate } from '@/lib/storage';
import { Header } from '@/components/Header';
import { InterviewBank } from './InterviewBank';

export const dynamic = 'force-dynamic';

export default async function InterviewPage() {
  await requireSubscription();

  const today = getTodayDate();

  // Flatten primer interviewQs into a typed array — no AI generation, static data only
  const questions = PRIMERS.flatMap(primer =>
    (primer.interviewQs ?? []).map(q => ({
      topic: primer.category,
      question: q.question,
      whatTheyWant: q.whatTheyWant,
      skeleton: q.skeleton,
    }))
  );

  return (
    <>
      <Header date={today} />
      <main className="min-h-screen bg-paper">
        <InterviewBank questions={questions} />
      </main>
    </>
  );
}
```
