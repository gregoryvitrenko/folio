<objective>
Fix story card overflow at 375px by adding min-w-0 to grid item wrappers.

Purpose: CSS Grid assigns `min-width: auto` to grid items by default. This means a grid item will never shrink below its content's intrinsic width — if any child (long URL, unbreakable firm name, long headline word) is wider than the viewport column, the card overflows horizontally. Adding `min-w-0` (which sets `min-width: 0`) to the grid item wrappers overrides this default and allows the grid columns to compress correctly to the viewport width.

Two wrappers need the fix:
- The `<div role="link">` in `StoryCard.tsx` — the outermost element of each card
- The `<div id="story-{id}">` in `StoryGrid.tsx` — the direct grid child that wraps StoryCard

Output: Cards lay out within viewport width on mobile without horizontal overflow.
</objective>
